    About the project

    The project follows the question:

    How Might We solve the housing problem for individuals embarking on their professional journey (entering the workforce) in Western Australia?

    Together with marketing students from Edith Cowan University in Perth, Australia, we created a concept which my group (acting design team) would design and build and they (acting marketing team) would market and sell.

    Creating an identity

    I took the lead to create the first graphic profile for this project and set the identity for this application.

    First, a few keywords were set up, schedule, synchronize, professional, housing, and FIFO-workers. Out of all these I chose to continue with schedule and synchronize for the logo.

    Since we already had the name Staysync from our previous sessions with ECU, the creation of the logotype worked out quite like this:

    For the color palette, we wanted to use something modern and clean. Taking inspiration from other professional apps, such as bank-apps, we chose the trendy kind of blurple color and followed up with neutral colors to adapt to the 60-30-10 rule.

    The logo and the color palette was then paired with a modern, readable font that allows some playfullness without disrupting that design-professional balance. Putting all of that together made our graphic profile which isn't very exstensive due to the time frames to work with, but it was nonetheless helpful to go back to when designing.

    Development

    The development was a smaller portion of this course and project - and even more so for my group which was most design focused. Even though I put most time into designing I also took the heavy load of the development since I was the most experienced within React which we were supposed to use for developing.

    To fit the development into the time frame the parts to be developed were discussed with our teacher and only a selected number of things were coded. Those things were rental ads feed, rental ads pages (when clicking in on an ad), filter and search, profile, navigation, and login. In this I had the responsibility for and developed the feed, rendal ads page, filter and search, database connection and design for login and outside of our list I had the main responsibility for the backed for which we used Firebase.

    React Native Expo

    The development was done using React Native through Expo connected to a github repository for easy version control and navigation.

    Backend

    The backend was handled using Firebase through authentication and firestore. The Firebase Authentication handled the email-login (which in reality would have been Australias version of Bank-ID) and Firestore was used to manually add ads and their info (since admin side where ads are posted was not a part of this project). The firestore collection was then read into a flatlist in the react project for the feed.

    The filtering and search was done looking for matching words or boolean values towards firebase. In the ad-page, ID was used to fetch and display more information which also used image mapping for icon display.
